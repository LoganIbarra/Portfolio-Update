Cooking With Roux

Role: UX Research and Voice Interface Design

Client: CareerFoundry UX Design (Academic)

Timeline: 3 weeks

Tools: Figma, Alexa Developer Console, Miro, Google Sheets

FileView the final voice script for Cooking With Roux

Designed an Alexa skill that allowed users to find and cook recipes with voice navigation only.

Considerations: Some features were written into sample dialogues that were not written as part of the final scripts due to time limitations.

Assumptions - Due to time limitations, assumptions were made for the following:

  • Users of the app would be for the creation of a persona, Morgan
  • Speaking style of users for the creation of the system persona, Roux
  • General user needs prior to interviews

Define

I first established both my user persona, Morgan and my system persona, Roux to reference throughout the project.

User needs and stories were defined, focusing on key functionalities someone like Morgan might need during daily interactions with Roux. Some examples of user needs and stories were:

Provide recipes for breakfast, lunch, dinner, and snacks:

  • As a busy professional, I want to search for simple breakfast recipes on the weekend so I can maximize my free time
  • As a user, I want to find recipes that can be utilized for meal prep or leftovers for lunches so I can save time getting ready in the morning
  • As Morgan, I want to find new dinner recipes to prepare so I don’t keep falling back to my same, easy meals after a long day
  • When I am driving home from work, I want to be provided dinner recipe options so I can stop by the store and grab ingredients if necessary

Use the recipe while cooking:

  • While preparing my meal, I want ingredients listed to me in groupings so that I don’t get overwhelmed or have to listen to long lists of items
  • As Morgan, I want instructions provided in a sequential, efficient fashion so that I feel confident I’m doing things correctly
  • As a user, I want the option to repeat any instructions in the event I need clarification or don’t hear it properly
  • As a person who is multitasking, I want to be the one to push the instructions forward when I’m ready so I can prepare the meal at my own pace
  • As a user, I want clear, descriptive instructions so I can more easily understand what I’m trying to achieve

I then wrote sample dialogues between our user and the system persona Roux. This would help me consider future design choices and necessary functionalities. A selection of sample dialogues included:

As a busy professional, I want to search for simple breakfast recipes on the weekend so I can maximize my free time.

  • U: Recommend a recipe for breakfast.
  • S: Certainly! We can prepare a tuna frittata, a hearty egg dish with potatoes and dollops of sour cream baked in. Cook time is 25 minutes. How does this sound?
  • U: Can you recommend something faster?
  • S: Of course. I have a recipe for avocado toast, topped with a fried egg and sprouts. Cook time is 10 minutes. Say ‘Get Started’ or ‘More Options’.
  • U: Get started.

As a user, I want the option to repeat any instructions in the event I need clarification or don’t hear it properly.
User is progressing through a recipe with the help of the assistant.

  • S: Step two, add diced potatoes to heated pan with olive oil and cook until they start to brown.
  • U: (Alt 1) Repeat.
  • U: (Alt 2) Repeat that.
  • U: (Alt 3) Repeat step two.
  • S: Step two, add diced potatoes to heated pan with olive oil and cook until they start to brown.
  • User completes step and is ready to proceed.
  • U: (Alt 1) Next.
  • U: (Alt 2) Next step.

Expanding on my sample dialogues, I considered what a full experience between our user Morgan and Roux might look like with multiple requests.

Using this, I established a user flow for Cooking With Roux. View the user flow here.

With a full user flow, I wrote a script including utterances and prompts that could be used for the Alexa Developer Console. I further iterated on this using proactive information strategies and tapered prompts.

User Testing

I set up a Wizard of Oz usability test and recruited participants. The participant would communicate with me directly and I would respond to them as Roux within the confines of the voice script written. I wrote a test plan and established objectives.

Test Objectives

Do participants want to find and cook new recipes?

What utterances are commonly used when trying to find a new recipe?

Does the participant experience any pain points during the interaction of trying to find a recipe?

Observe participants’ reactions/interactions with the system persona name of "Roux".

Method

Five (5) moderated in-person “Wizard of Oz” style usability tests. Included a short briefing, tasks, and a debriefing.

Participants were asked to measure the ease of each task using the single ease 7-point rating scale.

Audio transcripts were compiled and I synthesized the data into an affinity map. View the affinity map here.

Using the affinity map, I created a rainbow spreadsheet to assist with pinpointing key areas for improvement. You can view the rainbow spreadsheet here.

I updated my voice scripts based on the testing results. This included adding utterances for more polite verbiage such a "may I", adding instructions for how to handle compound ingredient requests, and instructions for error handling that keeps users progressing.

Reflect

This was a fantastic project to familiarize myself voice user interface design principles as well as developing for systems such as Alexa. If I had more time, I would have preferred to research the user needs and speaking styles as opposed to making assumptions. It was a good experience to move on decisions in order to progress and investigate through testing shortly thereafter.
