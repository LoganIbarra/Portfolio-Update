Cooking With Roux

Role: UX Research and Voice Interface Design

Client: CareerFoundry UX Design (Academic)

Timeline: 3 weeks

Tools: Figma, Alexa Developer Console, Miro, Google Sheets

Cooking With Roux is an Alexa Skill that allows users to find and cook recipes through voice navigation only. Due to time limitations, sample dialogues contain features that weren't a part of the final script. This project exhibits my ability to work with limited information, testing and iterating to make progress. The skill combines my interests in research, design, and cooking, all in one place.

Define

Assumptions - Due to time limitations, assumptions were made for the following:

  • Users of the app would be for the creation of a persona, Morgan
  • Speaking style of users for the creation of the system persona, Roux
  • General user needs prior to interviews

I established my user persona, Morgan, and my system persona, Roux, to reference throughout the project.

User needs and stories were defined, focusing on primary functionalities someone like Morgan might need during daily interactions with Roux. Some examples of user needs and stories were:

Provide recipes for breakfast, lunch, dinner, and snacks:

  • As a busy professional, I want to search for simple breakfast recipes on the weekend to maximize my free time.
  • As a user, I want to find recipes that can be utilized for meal prep or leftovers for lunches so I can save time getting ready in the morning.
  • As the user Morgan, I want to find new dinner recipes to prepare; so I don’t fall back on the same meals after a long day.
  • When I am driving home from work, I want dinner recipe options so I can stop by the store and grab ingredients if necessary.

Use the recipe while cooking:

  • While preparing my meal, I want ingredients listed in groupings so that I don’t get overwhelmed or have to listen to long lists of items.
  • As the user Morgan, I want instructions provided in a sequential, efficient fashion to feel confident I’m doing things correctly.
  • As a user, I want the option to repeat any instructions if I need clarification or don’t hear it properly.
  • While multitasking, I want to be the one to push the instructions forward so I can prepare the meal at my own pace.
  • As a user, I want clear, descriptive instructions where I can easily understand what I’m trying to achieve.

I then wrote sample dialogues between our user and the system persona Roux. The sample dialogues would help me consider future design choices and necessary functionalities.

As a busy professional, I want to search for simple breakfast recipes on the weekend to maximize my free time.

  • U: Recommend a brakfast recipe.
  • S: Certainly! We can prepare a tuna frittata, a hearty egg dish with potatoes and dollops of sour cream baked in. Cook time is 25 minutes. How does this sound?
  • U: Can you recommend something faster?
  • S: Of course. I have a recipe for avocado toast, topped with a fried egg and sprouts. Cook time is 10 minutes. Say ‘Get Started’ or ‘More Options.’
  • U: Get started.

Expanding on my sample dialogues, I considered what a complete experience between our user Morgan and Roux might look like with multiple requests.

Using this, I established a user flow for Cooking With Roux. View the user flow here.

With the user flow, I wrote a script including utterances and prompts that I would transfer to the Alexa Developer Console. I further iterated on this using proactive information strategies and tapered prompts.

User Testing

I set up a Wizard of Oz usability test and recruited participants. The participant would communicate with me directly, and I would respond to them as Roux within the confines of the voice script written. I wrote a test plan and established objectives.

Test Objectives

Do participants want to find and cook new recipes?

What are commonly used utterances when trying to find a new recipe?

Does the participant experience any pain points when trying to find a recipe?

Observe reactions/interactions with the system persona name.

Method

Five (5) moderated in-person “Wizard of Oz” style usability tests that included a short briefing, tasks, and a debriefing.

Participants measured the ease of each task using the single ease 7-point rating scale.

I compiled audio transcripts and synthesized the data into an affinity map. View the affinity map here.

Using the affinity map, I created a rainbow spreadsheet to assist with pinpointing areas for improvement. You can view the rainbow spreadsheet here.

Based on the testing results, I added utterances for more polite wording such as "may I," provided instructions for handling compound ingredient requests, and error instruction that keeps users progressing.

Reflect

This project familiarized me with voice interface design principles and developing for systems such as Alexa. If I had more time, I would have preferred to research the user's needs and speaking styles instead of making assumptions. It was a positive experience to make decisions to progress and investigate through testing shortly after.
